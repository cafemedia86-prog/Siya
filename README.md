# Siya: The Flavors of India 🌶️☕

Siya is a premium B2B e-commerce platform dedicated to the global export of Indian spices and tea. This platform bridges the gap between Indian heritage and global markets, offering a seamless experience for bulk buyers and exporters.

## ✨ Key Features

- **Premium UI/UX**: A modern, responsive design using Tailwind CSS 4 and fluid animations with Motion.
- **Product Catalog**: Explore a curated selection of premium Indian spices and teas.
- **Wholesale Inquiries**: Dedicated system for bulk buyers to submit inquiries and receive quotes.
- **Admin Dashboard**: Comprehensive management interface for products, categories, and wholesale inquiries.
- **Dark Mode Support**: Seamless theme switching for a premium user experience.
- **Search & Filtering**: Advanced search capabilities to quickly find specific spice or tea varieties.

## 🛠️ Technology Stack

- **Frontend**: [React 19](https://react.dev/)
- **Build Tool**: [Vite 6](https://vitejs.dev/)
- **Styling**: [Tailwind CSS 4](https://tailwindcss.com/)
- **Animations**: [Motion](https://motion.dev/)
- **Icons**: [Lucide React](https://lucide.dev/)
- **Routing**: [React Router 7](https://reactrouter.com/)
- **Backend/Stub**: [Express](https://expressjs.com/) & [Better-SQLite3](https://github.com/WiseLibs/better-sqlite3) (for data handling)

## 🚀 Getting Started

### Prerequisites

- [Node.js](https://nodejs.org/) (Latest LTS recommended)
- [npm](https://www.npmjs.com/)

### Installation

1. **Clone the repository:**
   ```bash
   git clone https://github.com/cafemedia86-prog/Siya.git
   cd siya
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Configure Environment Variables:**
   Create a `.env` file in the root directory based on `.env.example`.

4. **Start the development server:**
   ```bash
   npm run dev
   ```
   The application will be available at `http://localhost:3000`.

## 📦 Project Structure

- `src/components`: Reusable UI components (Navbar, Hero, Product Cards, etc.).
- `src/pages`: Main application views (Home, Products, Admin Dashboard, etc.).
- `src/context`: Application-wide state management (Admin context, etc.).
- `src/lib`: Utility functions and shared library configurations.
- `src/types.ts`: TypeScript interfaces and type definitions.
